# My DevOps Project

Демонстрационный проект, реализующий контейнеризированное веб-приложение на Flask с интегрированным стеком мониторинга (Prometheus + Grafana). Основная цель — отработка практик упаковки приложений, оркестрации сервисов и настройки observability.

## Стек технологий
- **Приложение**: Python 3.11, Flask, Gunicorn
- **Контейнеризация**: Docker (Multi-stage build), Docker Compose
- **Мониторинг**: Prometheus, Grafana, prometheus-flask-exporter

## Архитектура и ключевые решения

1. **Multi-stage сборка Docker**  
   Финальный образ собирается на базе `python:3.11-alpine`. Зависимости устанавливаются на промежуточном этапе (`slim` образ), что позволяет исключить из финального образа компиляторы и исходники, сокращая его размер и уменьшая поверхность для атак.

2. **WSGI-сервер Gunicorn**  
   Вместо встроенного dev-сервера Flask используется Gunicorn с конфигурацией воркеров. Это стандартный подход для обеспечения конкурентной обработки запросов в production-среде.

3. **Сетевая изоляция**  
   Все сервисы объединены в пользовательскую Docker-сеть `monitoring`. Это обеспечивает встроенный DNS-резолвинг (сервисы обращаются друг к другу по именам контейнеров, например, `http://prometheus:9090`) и изолирует их от хост-машины.

4. **Сбор метрик**  
   Интеграция выполнена через `prometheus-flask-exporter`. Он автоматически инструментирует маршруты Flask, отдавая метрики (количество запросов, задержки) на эндпоинте `/metrics` без изменения бизнес-логики приложения.

## Требования
- Установленные Docker и Docker Compose (v2.x)
- Свободные порты: 5000, 9090, 3000

## Запуск

1. Клонировать репозиторий и перейти в директорию:
   ```bash
   git clone https://github.com/Ghostosmoke/my_devops_project.git
   cd my_devops_project
   ```

2. Собрать образы и запустить сервисы в фоновом режиме:
   ```bash
   docker compose up -d --build
   ```

3. Проверить статус контейнеров:
   ```bash
   docker compose ps
   ```

## Проверка работы

После запуска сервисы доступны по следующим адресам:

| Сервис | URL | Примечание |
| :--- | :--- | :--- |
| **Веб-приложение** | `http://localhost:5000` | Главная страница |
| **Health Check** | `http://localhost:5000/health` | Возвращает `200 OK` (готово для k8s Liveness Probe) |
| **Метрики** | `http://localhost:5000/metrics` | Формат Prometheus |
| **Prometheus** | `http://localhost:9090` | UI для PromQL запросов |
| **Grafana** | `http://localhost:3000` | Логин/пароль по умолчанию: `admin` / `admin` |

*Для настройки Grafana:* добавьте Data Source типа Prometheus с URL `http://prometheus:9090`. После этого можно строить дашборды на основе метрик, например, `flask_http_request_total`.
